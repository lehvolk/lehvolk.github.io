Why Quantile?

There are two main projects for statistical computing:
  • R programming language
  • Julia programming language
There are mathematical packages like Octave, MatLab, Maple, Mathematica and others.

There is one common thing - all of them provides new programming language with interpreter.

This brings overhead for package:
  • some functions/structures is written with this language and each call of such function brings new work for slow interpreter
  • each time package developers implements instruments such as implementation of JIT, multi-threading, http protocol
Some packages are not open-source, some are paid software. Quantile is intended to solve this problem:

1) Quantile uses Scala programming language with rich possibilities (available api for working with data structures and is great).

2) Code runs in fast environment of JVM with built-in JIT compiler.

3) Easy using of multi-threading paradigm from Scala/Akka framework.

4) All Scala/Java instruments can be used:
  • connectors to databases (SQL and NoSQL)
  • ability to use enterprise clusters (from Java specific clusters such Hazelcast to BigData)
  • tons of frameworks for working with json, xml, low-level protocols and others

5) Scala brings ability to write code in Object Oriented or Functional style.

6) All solutions are well tested in enterprise projects.

7) Software is cross-platform so it can be run on all platforms where JVM can e run.

8) Best development environments (IDEA and Eclipse) with built-in GUI interface

All this benefits have an revert size:

1) JVM consumes more hardware resources from then standard package environment

2) every time you run code Scala compiles your script into JVM bytecode. Rich possibilities of Scala language make this compiler slow. So developer doesn't see speed increase on simple or atomic operations.
